# 法律时光机 MVP 文档

> 「先看结局，再决定要不要开始。」

## 1. 产品定位

AI 驱动的诉讼决策辅助工具。用户上传案情信息和证据，AI 模拟推演诉讼过程，给出胜率预测、成本分析和策略建议，帮助用户做出「要不要打官司」的决策。

**目标用户：** 有法律纠纷但不确定是否值得诉讼的普通人（非律师）

---

## 2. MVP 核心功能（4 个）

### 2.1 案情录入

- 用户选择纠纷类型（民事/刑事/合同/劳动/其他）
- 文本框描述案情经过（≥ 50 字）
- 上传证据文件（图片、PDF、合同照片，最多 5 份）
- 填写标的金额（元）

### 2.2 胜率预测仪表盘

- AI 分析案情 + 证据，输出胜诉概率（0-100%）
- 环形仪表盘可视化展示
- 胜率区间标签：
  - 🟢 80-100%：建议起诉
  - 🟡 50-79%：需补充证据
  - 🔴 0-49%：建议和解
- 简要说明判断依据（引用具体法条）

### 2.3 证据补强清单

- 分析现有证据的完整度
- 列出「缺失证据」清单（如：缺少书面合同、缺少转账记录、缺少聊天记录截图）
- 每项标注优先级（关键 / 重要 / 辅助）
- 给出取证建议（去哪里获取、怎么保全）

### 2.4 成本收益计算器

- 自动计算预估成本：
  - 诉讼费（按标的额阶梯计算）
  - 律师费（按地区均价估算）
  - 时间成本（预估周期：3-12 个月）
- 预估收益 = 标的额 × 胜率
- 净收益 = 预估收益 - 总成本
- 结论建议：✅ 值得诉讼 / ⚠️ 谨慎考虑 / ❌ 建议和解

## 4. 技术方案

### 4.1 技术栈

| 层级 | 方案 | 说明 |
|------|------|------|
| 前端 | Vue 3 + Vite + TailwindCSS | 快速开发，移动端优先 |
| 后端 | Node.js + Express | 轻量 API 服务 |
| AI | 通义千问 / OpenAI API | 案情分析、胜率推理 |
| 文件存储 | 本地 / OSS | MVP 阶段本地存储 |
| 部署 | Docker + Nginx | 单机部署即可 |

### 4.2 API 设计

```
POST /api/analyze
Content-Type: multipart/form-data

Body:
  - caseType: string       // 纠纷类型
  - description: string    // 案情描述
  - amount: number         // 标的金额
  - files: File[]          // 证据文件

Response:
{
  "winRate": 78,
  "winRateLevel": "medium",         // high / medium / low
  "judgmentBasis": "根据《民法典》第XXX条...",
  "costBreakdown": {
    "courtFee": 2300,
    "lawyerFee": 8000,
    "timeCost": "4-6个月",
    "totalCost": 10300
  },
  "expectedReturn": 39000,
  "netReturn": 28700,
  "recommendation": "worth",        // worth / cautious / settle
  "missingEvidence": [
    {
      "name": "书面合同原件",
      "priority": "critical",       // critical / important / helpful
      "suggestion": "联系对方补签，或查找邮件/微信中的合同确认记录"
    }
  ],
  "opponentDefense": [
    {
      "strategy": "主张合同无效",
      "likelihood": "high",
      "countermeasure": "准备合同签署时的录音或见证人证言"
    }
  ]
}
```

### 4.3 AI Prompt 设计（核心）

```
你是一名资深中国律师。请根据以下案情信息，进行诉讼推演分析。

【案情类型】{{caseType}}
【案情描述】{{description}}
【标的金额】{{amount}}元

请以 JSON 格式输出：
1. winRate: 胜诉概率 (0-100整数)
2. judgmentBasis: 判断依据（引用具体法条）
3. missingEvidence: 缺失证据清单（含优先级和取证建议）
4. opponentDefense: 对方可能的抗辩策略（含应对方案）
5. estimatedDuration: 预估诉讼周期
```

## 7. 风险与对策

| 风险 | 对策 |
|------|------|
| AI 分析不准确，用户盲目信任 | 明确标注「仅供参考，不构成法律意见」 |
| 用户上传敏感文件 | 文件加密存储，分析完成后 24h 自动删除 |
| 法律法规变化 | Prompt 中引用最新法条，定期更新知识库 |
| 恶意上传非法内容 | 内容审核 + 上传限制 |
